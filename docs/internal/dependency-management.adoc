= Dependency management Structure

== Overview

This document describes the Gradle plugin structure used in the project,
with a focus on centralized dependency management. The goal is to keep
dependency versions consistent across all modules, avoid "jar hell",
and ensure stable artifacts.

== Plugins

=== shared.gradle.kts
* Applied in all modules (`common`, `sbs`, and `master`).
* Defines dependency management for:
** Spring Boot BOM (`org.springframework.boot:spring-boot-dependencies`)
** Spring Cloud BOM (`org.springframework.cloud:spring-cloud-dependencies`)
** Other shared libraries (e.g., JSpecify).
* Guarantees consistent and compatible versions across all the components.

=== common.gradle.kts
* Applied in all `common` modules.
* Extends the `shared` plugin.
* Contains configurations specific to common modules.

=== sbs.gradle.kts
* Applied in Spring Boot Starter (`sbs`) modules.
* Extends the `shared` plugin.
* Adds additional dependencies required for starter modules.

=== master (build.gradle.kts)
* Belongs to the root `master` module (not split into submodules).
* Also applies the `shared.gradle.kts` plugin to ensure consistent dependency management.
